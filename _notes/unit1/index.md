## 1. What is Reinforcement Learning?

**Definition:** Reinforcement Learning (RL) is a type of machine learning where an **agent** learns to make decisions by interacting with an **environment**. The agent receives **rewards** (positive or negative) based on its actions and aims to maximize the total reward over time.

**Two unique characteristics of RL:**
1. **Evaluative feedback** — the agent is told only how good its chosen action was, not which action would have been correct (unlike supervised learning).
2. **Sequential decision-making** — actions affect not just the immediate reward but also future states and rewards.

**How RL differs from supervised learning:**

| Feature | Supervised Learning | Reinforcement Learning |
|---|---|---|
| Feedback | Correct label given | Only reward for action taken |
| Data source | Fixed labelled dataset | Generated by agent–environment interaction |
| Goal | Learn a mapping input → output | Learn a policy to maximize long-run reward |

---

### 1.1 Key Elements of Reinforcement Learning

Every RL problem is described by six elements:

1. **Agent** — the learner/decision-maker (e.g., a robot, a game-playing AI).
2. **Environment** — everything outside the agent (e.g., the physical world, a game engine).
3. **State** ($S_t$) — the current situation of the environment observed by the agent.
4. **Action** ($A_t$) — a choice the agent makes at each step.
5. **Reward** ($R_t$) — a scalar signal the environment sends the agent after each action. It encodes what the agent should maximize.
6. **Policy** ($\pi$) — the agent's strategy; a mapping from states to actions. The goal is to find the optimal policy.

### 1.2 The Agent–Environment Interaction Loop

At each time step $t$:
1. Agent observes state $S_t$.
2. Agent selects action $A_t \sim \pi(\cdot \mid S_t)$.
3. Environment produces reward $R_{t+1}$ and transitions to $S_{t+1}$.
4. Cycle repeats.

![Agent–environment interaction loop]({{ site.baseurl }}/notes/unit1/images/image_05_agent_environment_loop.png)

**Real-world applications of RL:** game playing (Chess, Go), robotics, recommendation systems, autonomous driving, drug discovery, network routing, dynamic pricing.

**Limitations in practice:** requires many interactions to learn (sample inefficiency), reward design is hard (reward hacking), safety concerns during exploration, does not generalize well across tasks.

## 2. The Multi-Armed Bandit Problem

**Definition:** The multi-armed bandit (MAB) is a simplified RL problem where an agent repeatedly chooses among $k$ actions ("arms") and receives a reward, but there is **no state** — the same decision situation repeats every round.

The name comes from a bank of slot machines ("one-armed bandits"), each paying out at a different unknown rate.

![Slot machines illustrating the k-armed bandit]({{ site.baseurl }}/notes/unit1/images/image_03_slot_machines_row.png)

### 2.1 Formal Setup

- $k$ arms, $n$ total rounds.
- Each arm $a$ has a true mean reward $q_*(a) = \mathbb{E}[R_t \mid A_t = a]$.
- The **optimal arm** is $a^* = \arg\max_a q_*(a)$.
- The agent maintains an **estimate** $Q_t(a) \approx q_*(a)$.
- At each round: select arm $A_t$, observe reward $R_t$, update estimate.

**Key notation (used throughout):**

| Symbol | Meaning |
|---|---|
| $q_*(a)$ | True mean reward of arm $a$ |
| $Q_t(a)$ | Agent's estimate of $q_*(a)$ at time $t$ |
| $N_t(a)$ | Number of times arm $a$ has been pulled |
| $\Delta_a = q_*(a^*) - q_*(a)$ | Suboptimality gap of arm $a$ |

### 2.2 MAB vs Full RL

| Aspect | Multi-Armed Bandit | Full RL |
|---|---|---|
| State | None (stateless) | State changes over time |
| Actions affect future | No | Yes |
| Goal | Find best arm | Find best policy over states |
| Feedback | Evaluative (immediate) | Evaluative (possibly delayed) |

**Why study bandits?** They isolate the exploration–exploitation problem in its purest form. Solutions developed here (UCB, Thompson Sampling) extend directly to full RL and are widely used in industry (A/B testing, recommendation systems, clinical trials).

---

## 3. Exploration vs Exploitation

**Definition:** The fundamental tension in RL and bandits between:
- **Exploitation** — choose the arm with the highest *current* estimate (greedy). Maximizes immediate reward but may miss better arms.
- **Exploration** — try arms with uncertain estimates to gather information. Sacrifices immediate reward to find better arms.

![The exploration–exploitation fork]({{ site.baseurl }}/notes/unit1/images/image_06_exploration_exploitation_fork.png)

Both are necessary: pure exploitation gets stuck on suboptimal arms; pure exploration never capitalizes on what it learns.

## 4. Greedy Policy and ε-Greedy Algorithm

### 4.1 Greedy Policy

**Definition:** An agent follows a **greedy policy** when it always selects the arm with the highest estimated value:

$$A_t = \arg\max_a Q_t(a)$$

**Problem:** If the best arm gets one unlucky early reward, the agent may never revisit it. Pure greedy has **no exploration**.

### 4.2 Sample-Average Estimation

The action-value estimate is the running average of observed rewards:

$$Q_t(a) = \frac{\text{sum of rewards when } a \text{ was chosen}}{\text{number of times } a \text{ was chosen}} = \frac{\sum_{s<t} R_s \cdot \mathbf{1}[A_s = a]}{N_t(a)}$$

By the Law of Large Numbers, $Q_t(a) \to q_*(a)$ as $N_t(a) \to \infty$.

### 4.3 Incremental Update Rule

Computing the average from scratch each time takes $O(n)$ memory. A simple algebraic rewrite gives a constant-memory online update:

$$Q_{n+1}(a) = Q_n(a) + \frac{1}{n}\bigl[R_n - Q_n(a)\bigr]$$

**General form:** $\text{NewEstimate} \leftarrow \text{OldEstimate} + \alpha\,[\text{Target} - \text{OldEstimate}]$

- $\alpha = 1/n$: sample average (equal weight to all observations).
- $\alpha = $ constant: **exponential recency-weighted average** — forgets old data. Use for **nonstationary** problems where true means drift over time.

The term $[R_n - Q_n(a)]$ is an **error signal**: positive means the reward exceeded expectations; negative means it fell short.

### 4.4 ε-Greedy Algorithm

**Algorithm:**
- With probability $\varepsilon$: select a **random** arm (explore).
- With probability $1-\varepsilon$: select $\arg\max_a Q_t(a)$ (exploit).

Every arm gets pulled infinitely often, so all estimates converge to true means. The trade-off:
- Small $\varepsilon$ → slow learning but better long-run exploitation.
- Large $\varepsilon$ → fast learning but persistent random exploration (wastes pulls).
- **Decaying $\varepsilon$** (e.g., $\varepsilon_t = 1/t$) gets the best of both.

![Greedy vs ε-greedy performance on the 10-armed testbed]({{ site.baseurl }}/notes/unit1/images/sb_fig_2_2_greedy_vs_eps.jpeg)

*Figure: On a 10-armed Gaussian testbed, pure greedy plateaus at ~60% optimal; ε = 0.1 converges faster; ε = 0.01 eventually surpasses both. Greedy gets stuck on a suboptimal arm after unlucky early samples.*

<div class="problem">

**Worked Example.** An arm has estimate $Q_5 = 2.0$ after 4 pulls. The 5th reward is $R_5 = 5.0$. Compute $Q_6$.

**Sample average:** $Q_6 = 2.0 + \frac{1}{5}[5.0 - 2.0] = 2.0 + 0.6 = 2.6$

**Constant $\alpha = 0.1$:** $Q_6 = 2.0 + 0.1[5.0 - 2.0] = 2.0 + 0.3 = 2.3$

The sample average reacts more strongly; constant $\alpha$ gives more weight to established history — useful for stationary environments.

</div>

## 5. Optimistic Initial Values

**Definition:** Setting $Q_1(a)$ to an **optimistically high value** (well above what rewards can realistically be) so the agent is initially "disappointed" by every arm and is driven to explore all of them.

**How it works:** Whichever arm the agent pulls, its estimate drops toward the true mean. The agent then tries another arm (since that one now looks best). This cycle forces early systematic exploration without any explicit random exploration.

**Limitation:** Only helps at the beginning. Once all arms are pulled a few times, the effect fades. Not useful for nonstationary problems.

---

## 6. Cumulative Regret

**Definition:** Regret measures how much reward we lost by not always playing the optimal arm:

$$\text{Reg}(n) = n\,q_*(a^*) - \sum_{t=1}^{n} q_*(A_t) = \sum_{a} \Delta_a \cdot \mathbb{E}[N_n(a)]$$

where $\Delta_a = q_*(a^*) - q_*(a)$ is the **gap** of arm $a$.

**Key properties:**
- $\text{Reg}(n) \geq 0$ always (the oracle always does at least as well).
- $\text{Reg}(n) = 0$ only if we always pull the optimal arm.
- **Goal:** achieve **sublinear regret** — $\text{Reg}(n)/n \to 0$ as $n \to \infty$ (we play the right arm almost all the time eventually).

![Regret growth comparison: sublinear vs linear]({{ site.baseurl }}/notes/unit1/images/image_07_regret_growth_comparison.png)

**Regret decomposition insight:** each pull of a suboptimal arm $a$ costs $\Delta_a$ in expected reward. To minimize regret, pull arms with large $\Delta_a$ rarely and arms with $\Delta_a \approx 0$ freely. The challenge: we don't know $\Delta_a$ — we must estimate means and reason about uncertainty.

<div class="problem">

**Worked Example.** A 3-armed Bernoulli bandit: $q_*(1) = 0.3$, $q_*(2) = 0.5$, $q_*(3) = 0.8$. After $n = 1000$ rounds, a policy pulled each arm (400, 350, 250) times. Compute $\text{Reg}(1000)$.

$\Delta_1 = 0.8 - 0.3 = 0.5$, $\Delta_2 = 0.8 - 0.5 = 0.3$, $\Delta_3 = 0$.

$\text{Reg}(1000) = (0.5)(400) + (0.3)(350) + (0)(250) = 200 + 105 + 0 = 305$

</div>

## 7. Upper Confidence Bound (UCB)

### 7.1 Motivation: Optimism Under Uncertainty

**Principle:** act as if each arm is as good as it **plausibly could be** given the data. Arms that have been pulled few times deserve an **optimism bonus** — they might be great, we just haven't checked enough.

![UCB optimism: wide CI for rarely-pulled arms, narrow for well-sampled ones]({{ site.baseurl }}/notes/unit1/images/image_02_optimism_confidence_caps.png)

### 7.2 UCB1 Algorithm

At each round $t$, select:

$$A_t = \arg\max_a \left[ Q_t(a) + \sqrt{\frac{2 \ln t}{N_t(a)}} \right]$$

The term $\sqrt{2 \ln t / N_t(a)}$ is the **exploration bonus**:
- Large when $N_t(a)$ is small (few pulls → high uncertainty → explore).
- Shrinks as the arm is pulled more.
- Even ignored arms eventually get re-checked: as $t$ grows, $\ln t$ grows but $N_t(a)$ stays fixed, so the bonus creeps up.

**No ε to tune.** Exploration is automatic and data-driven.

**Regret bound:** $\mathbb{E}[\text{Reg}(n)] = O(\ln n)$ — dramatically better than ε-greedy's $O(n)$ in the worst case.

![UCB vs ε-greedy on the 10-armed testbed]({{ site.baseurl }}/notes/unit1/images/sb_fig_2_4_ucb_testbed.jpeg)

*Figure: UCB (c=2) outperforms all fixed-ε versions; it starts slower but converges to near-optimal behavior.*

<div class="problem">

**Worked Example.** Three arms at $t = 17$: Arm 1 pulled 10 times ($\hat{\mu} = 0.6$), Arm 2 pulled 4 times ($\hat{\mu} = 0.5$), Arm 3 pulled 2 times ($\hat{\mu} = 0.4$). Compute UCB1 indices.

$$\text{UCB}_1 = 0.6 + \sqrt{\frac{2\ln 17}{10}} = 0.6 + \sqrt{0.566} \approx 0.6 + 0.580 = 1.180$$
$$\text{UCB}_2 = 0.5 + \sqrt{\frac{2\ln 17}{4}} \approx 0.5 + 0.917 = 1.417$$
$$\text{UCB}_3 = 0.4 + \sqrt{\frac{2\ln 17}{2}} \approx 0.4 + 1.296 = 1.696$$

**UCB1 selects Arm 3** — despite its lowest sample mean, it has the widest uncertainty and highest upper bound.

</div>

### 7.3 UCB vs ε-Greedy: Key Comparison

| Aspect | ε-Greedy | UCB1 |
|---|---|---|
| Exploration | Random, fixed probability | Directed, uncertainty-driven |
| Tuning | Must set ε | No free parameter (uses $\ln t$) |
| Regret | $O(\varepsilon n)$ or $O(\sqrt{n})$ | $O(\ln n)$ |
| Handles nonstationary? | Yes (with decay) | Poorly (count-based) |
| Simple to implement? | Very simple | Simple |

## 8. KL-UCB (Brief)

**Motivation:** Plain UCB uses a symmetric confidence interval derived assuming worst-case variance ($\sigma = 1/2$). For **Bernoulli rewards** (click/no-click, win/loss) with small means (e.g., 2% CTR), this interval is too wide — it extends below zero (nonsense for a probability).

**Key idea:** Replace the symmetric bonus with the **KL divergence** between Bernoulli distributions. The index for arm $a$ is:

$$\text{index}_t(a) = \max\{q \geq \hat\mu_t(a) : d(\hat\mu_t(a),\, q) \leq \tfrac{\ln t}{N_t(a)}\}$$

where $d(p, q) = p\ln\frac{p}{q} + (1-p)\ln\frac{1-p}{1-q}$ is the Bernoulli KL divergence.

**Result:** tighter, asymmetric intervals → fewer wasted pulls on clearly-bad arms → matches the Lai–Robbins lower bound exactly for Bernoulli rewards.

**When to use:** binary rewards (ads, clicks, conversions), especially at low rates.

## 9. Thompson Sampling

### 9.1 Core Idea

Instead of computing a deterministic upper bound, Thompson Sampling (TS) maintains a **probability distribution over each arm's true mean** and selects arms by **sampling** from those beliefs.

Steps:
1. Maintain a **posterior belief** $P(\mu_a \mid \text{data})$ for each arm.
2. At each round: **draw one sample** $\theta_a$ from each arm's posterior.
3. **Play** the arm with the highest sample: $A_t = \arg\max_a \theta_a$.
4. Observe reward and **update** the posterior.

### 9.2 Beta Distribution for Bernoulli Arms

For Bernoulli rewards, the conjugate prior is the **Beta distribution**. Each arm is described by two counters:

- $\alpha_a = $ (wins so far) + 1
- $\beta_a = $ (losses so far) + 1

The posterior is $\text{Beta}(\alpha_a, \beta_a)$ with:
- Mean: $\mu = \alpha/(\alpha + \beta)$ — current best estimate.
- Variance: shrinks as data accumulates — uncertainty reduces.

![Beta posterior evolving as data accumulates for each arm]({{ site.baseurl }}/notes/unit1/images/plot_08_beta_belief_evolution.png)

*Figure: With few pulls, the Beta distribution is wide (high uncertainty). With more pulls, it concentrates around the true mean.*

### 9.3 Thompson Sampling Algorithm

```
Initialize: α[a] = 1, β[a] = 1  for all arms a    # uniform prior
for each round t:
    for each arm a:
        θ[a] ~ Beta(α[a], β[a])                    # sample from belief
    A_t = argmax_a θ[a]                             # play lucky arm
    observe reward R_t ∈ {0, 1}
    α[A_t] += R_t                                   # win: α up
    β[A_t] += 1 - R_t                              # loss: β up
```

No tuning constants. State = 2 integers per arm. Update = 2 additions per round.

![Thompson Sampling: wide belief leads to diverse samples, narrow belief concentrates]({{ site.baseurl }}/notes/unit1/images/plot_09_ts_cartoon.png)

*Figure: An arm with little data (wide posterior) occasionally "rolls high" and gets explored. As data accumulates, beliefs narrow and only genuinely good arms get selected.*

### 9.4 Why TS Explores Automatically

- **Wide belief** (few pulls) → samples scatter across $[0,1]$ → uncertain arm sometimes wins the draw → gets pulled.
- **Narrow belief** (many pulls) → samples cluster near true mean → only selected if it's genuinely best.
- Exploration fades naturally as data accumulates — no schedule needed.

**Regret:** TS matches the Lai–Robbins lower bound asymptotically (same as KL-UCB) — optimal for Bernoulli bandits.

## 10. Gradient Bandit Algorithms

### 10.1 Preferences vs Values

All previous methods estimated **action values** $Q_t(a)$ (expected reward) and acted on those estimates. Gradient bandits take a different approach: learn a **preference** $H_t(a)$ for each action.

![Action values vs action preferences: the key conceptual difference]({{ site.baseurl }}/notes/unit1/images/image_01_value_vs_preference.png)

- $Q_t(a)$ has an absolute numerical meaning (expected reward in reward units).
- $H_t(a) \in \mathbb{R}$ has only **relative** meaning — adding a constant to all preferences leaves the policy unchanged.

### 10.2 Softmax Policy

Actions are selected probabilistically via the **softmax** function:

$$\pi_t(a) = P(A_t = a) = \frac{e^{H_t(a)}}{\sum_{b} e^{H_t(b)}}$$

- Higher preference → higher probability, but never zero.
- Built-in exploration: all actions always have non-zero probability.
- Initially $H_1(a) = 0$ for all arms → uniform random selection.

![Softmax curves for different preference values]({{ site.baseurl }}/notes/unit1/images/image_03_softmax_curves.png)

### 10.3 Gradient Update Rule

After observing reward $R_t$ at step $t$:

$$H_{t+1}(A_t) = H_t(A_t) + \alpha\,(R_t - \bar{R}_t)(1 - \pi_t(A_t))$$
$$H_{t+1}(a) = H_t(a) - \alpha\,(R_t - \bar{R}_t)\,\pi_t(a) \quad \text{for } a \neq A_t$$

where $\bar{R}_t$ is the **running average of all past rewards** (the baseline).

**Intuition:**
- Reward **above baseline** → increase preference for selected action, decrease others.
- Reward **below baseline** → decrease preference for selected action, increase others.

### 10.4 The Role of the Baseline

![Effect of the baseline on gradient bandit performance]({{ site.baseurl }}/notes/unit1/images/sb_fig_2_5_gradient_bandit_baseline.jpeg)

*Figure: Without a baseline ($\bar{R}_t = 0$), when all rewards are uniformly positive, preferences drift arbitrarily — performance degrades. The baseline keeps the update signal meaningful.*

**Key fact:** The expected update equals the true gradient of expected reward regardless of what baseline is used. The baseline affects only the variance of the update, not its expected direction.

## 11. Contextual Bandits

### 11.1 Definition

A **contextual bandit** (also called associative search) extends the MAB by adding a **context** (feature vector) $x_t$ observed before each action. The reward depends on both the context and the action.

**Formal setting:**
- Each round: observe context $x_t \in \mathcal{X}$.
- Select action $A_t$ using a **policy** $\pi(x_t)$.
- Receive reward $R_t \sim P(\cdot \mid x_t, A_t)$.

![Nonassociative (standard) bandit vs contextual bandit]({{ site.baseurl }}/notes/unit1/images/image_07_nonassociative_vs_contextual.png)

### 11.2 The Coloured Slot Machines Example

Imagine several slot machines, each with a coloured light (red, green, blue). The colour changes each round and indicates which "task" the agent is in. The best arm **depends on the colour** — the agent must learn a mapping: colour → best arm.

![Coloured slot machines illustrating contextual bandits]({{ site.baseurl }}/notes/unit1/images/image_08_coloured_slot_machines.png)

### 11.3 Contextual Bandit vs Full RL

| Aspect | Contextual Bandit | Full RL (MDP) |
|---|---|---|
| Actions affect future state | No | Yes |
| Needs policy | Yes (context → action) | Yes (state → action) |
| Planning horizon | Single step | Multi-step |

**Bridge role:** Contextual bandits sit between pure bandits (no state) and full RL (state affects future). Most real industry deployments (news feeds, ad targeting, personalization) are contextual bandits because feedback is immediate and no long-horizon planning is needed.

## Summary Table

| Algorithm | Exploration Mechanism | Key Formula / Idea | Best For |
|---|---|---|---|
| Greedy | None | $\arg\max Q_t(a)$ | Not recommended alone |
| ε-Greedy | Random with prob ε | $\varepsilon$-random, $(1-\varepsilon)$-greedy | Simple baseline |
| Optimistic Init | Forced early pulls | High $Q_1(a)$ | Stationary problems |
| UCB1 | Confidence bonus | $Q_t + \sqrt{2\ln t/N_t}$ | Generic rewards |
| KL-UCB | KL-tight bonus | $d(\hat\mu, q) \leq \ln t / N_t$ | Binary/Bernoulli rewards |
| Thompson Sampling | Posterior sampling | $\theta_a \sim \text{Beta}(\alpha_a, \beta_a)$ | Bernoulli, simple to ship |
| Gradient Bandit | Softmax over preferences | $H_t$ update via reward baseline | No value estimate needed |
| Contextual Bandit | Any of the above + context | Policy $\pi(x_t) \to a$ | Context-dependent rewards |
