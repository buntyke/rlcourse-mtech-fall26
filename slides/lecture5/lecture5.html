Recap: What We've Done with Values So Far
  • Callback: \(Q_t(a)\) drove every selection rule — greedy, \(\epsilon\)-greedy, UCB, KL-UCB, Thompson Sampling.
  • All previous methods estimate value, then act.
  • New idea today: skip the value estimate; learn a preference for each action directly.

\(\pi_t(a) \doteq \mathbb{P}[A_t = a]\)

Why Learn Preferences Instead of Values?
  • Action values have a fixed numerical meaning (expected reward).
  • Preferences \(H_t(a) \in \mathbb{R}\) carry no absolute meaning — only relative differences matter.
  • Adding a constant to every preference leaves the policy unchanged.
  • This freedom enables a clean gradient-based update.
Softmax (Gibbs / Boltzmann) Distribution

\(\mathbb{P}[A_t = a] \doteq \dfrac{e^{H_t(a)}}{\sum_{x=1}^{k} e^{H_t(x)}} \doteq \pi_t(a)\)

  • Initially \(H_1(a) = 0\) for all \(a\) — uniform policy.
  • Larger preference \(\Rightarrow\) higher selection probability, but never zero.
  • Built-in exploration: every action keeps non-zero probability.
Question #1 — Softmax = Sigmoid for k = 2
Show that for two actions, the softmax distribution reduces to the logistic / sigmoid function.

Hint: divide numerator and denominator by \(e^{H_t(1)}\) and let \(x = H_t(2) - H_t(1)\).
\(\pi_t(2) = \frac{e^{H_t(2)}}{e^{H_t(1)} + e^{H_t(2)}} = \frac{1}{1 + e^{-(H_t(2)-H_t(1))}} = \sigma(x)\).
Gradient Bandit in One Sentence
Gradient bandit: maintain a real-valued preference \(H_t(a)\) for each action, sample actions from a softmax over these preferences, and update the preferences after each step using a stochastic-gradient rule that pushes preference up when reward beats a baseline and down when it falls short.
  • No \(Q\)-estimate is ever computed.
  • Update is differentiable in \(H\) — that's what makes it "gradient".
The Update Rule (Eq. 2.12)

\(H_{t+1}(a) \doteq \begin{cases} H_t(a) + \alpha\,(R_t - \bar R_t)\,(1 - \pi_t(a)) & a = A_t \\[6pt] H_t(a) - \alpha\,(R_t - \bar R_t)\,\pi_t(a) & a \neq A_t \end{cases}\)

  • \(\alpha > 0\): step size.
  • \(\bar R_t\): running average of past rewards (the baseline).
  • Reward above baseline \(\Rightarrow\) selected action's preference increases.
  • Non-selected actions move in the opposite direction.
Sutton & Barto, Reinforcement Learning: An Introduction, Ch. 2.8, Eq. 2.12.
The Baseline \(\bar R_t\)

\(\bar R_{t+1} = \bar R_t + \tfrac{1}{t}\,(R_t - \bar R_t)\)

  • \(\bar R_t\) is the incremental average of all rewards up to (but not including) time \(t\).
  • For nonstationary problems, replace \(\tfrac{1}{t}\) with a constant step size \(\alpha\).
  • Choice of baseline does not change the expected update — only its variance (we'll prove this).
Gradient Bandit Algorithm
initialize H(a) = 0  for all a
initialize R_bar = 0,  n = 0

for t = 1, 2, ..., T:

    pi(a) = exp(H(a)) / sum_b exp(H(b))    # softmax

    A_t  ~  Categorical( pi )              # sample action
    observe reward R_t

    for each arm a:
        if a == A_t:
            H(a) += alpha * (R_t - R_bar) * (1 - pi(a))
        else:
            H(a) -= alpha * (R_t - R_bar) * pi(a)

    n += 1
    R_bar += (R_t - R_bar) / n             # update baseline
Sutton & Barto, Ch. 2.8.

Question #2 — Effect of Subtracting the Baseline
Scenario: Every reward in your problem lies between +9 and +11 — large absolute values, small differences between arms.

(a) What does the algorithm look like with \(\bar R_t = 0\)?
(b) Why does subtracting \(\bar R_t \approx 10\) make learning more reliable?
Without baseline every (R−0) is large & positive → every selected action has its preference pushed up regardless of relative quality. With baseline near 10 the sign of the update reflects above-vs-below average — exactly what we want.
Why the Update Works — Step 1: The Goal

\(H_{t+1}(a) \doteq H_t(a) + \alpha\,\dfrac{\partial \mathbb{E}[R_t]}{\partial H_t(a)}\)

\(\mathbb{E}[R_t] = \sum_x \pi_t(x)\,q_*(x)\)

  • We want exact gradient ascent on the expected reward.
  • Problem: we don't know \(q_*(x)\).
  • Plan: rewrite the gradient as an expectation we can sample.
Step 2: Introduce a Baseline \(B_t\)

\(\dfrac{\partial \mathbb{E}[R_t]}{\partial H_t(a)} = \sum_x q_*(x)\,\dfrac{\partial \pi_t(x)}{\partial H_t(a)}\)

\(= \sum_x \big(q_*(x) - B_t\big)\,\dfrac{\partial \pi_t(x)}{\partial H_t(a)}\)

because \(\sum_x \dfrac{\partial \pi_t(x)}{\partial H_t(a)} = 0\)

🖊️ Board: probabilities sum to 1, so their gradient sums to 0 — the baseline term vanishes.
Step 3: Multiply and Divide by \(\pi_t(x)\)

\(\dfrac{\partial \mathbb{E}[R_t]}{\partial H_t(a)} = \sum_x \pi_t(x)\,(q_*(x) - B_t)\,\dfrac{\partial \pi_t(x)/\partial H_t(a)}{\pi_t(x)}\)

\(= \mathbb{E}\!\left[(q_*(A_t) - B_t)\,\dfrac{\partial \pi_t(A_t)/\partial H_t(a)}{\pi_t(A_t)}\right]\)

Choose \(B_t = \bar R_t\) and substitute \(\mathbb{E}[R_t \mid A_t] = q_*(A_t)\):

\(= \mathbb{E}\!\left[(R_t - \bar R_t)\,\dfrac{\partial \pi_t(A_t)/\partial H_t(a)}{\pi_t(A_t)}\right]\)

Step 4: The Softmax Derivative (Boardwork)
🖊️ Claim: \(\dfrac{\partial \pi_t(x)}{\partial H_t(a)} = \pi_t(x)\,\big(\mathbb{1}_{a=x} - \pi_t(a)\big)\)

Quotient rule on the softmax:

\(\dfrac{\partial}{\partial H_t(a)}\dfrac{e^{H_t(x)}}{\sum_y e^{H_t(y)}} = \dfrac{\mathbb{1}_{a=x}\,e^{H_t(x)}\sum_y e^{H_t(y)} \;-\; e^{H_t(x)}\,e^{H_t(a)}}{\left(\sum_y e^{H_t(y)}\right)^2}\)

\(= \mathbb{1}_{a=x}\,\pi_t(x) - \pi_t(x)\,\pi_t(a) = \pi_t(x)\,(\mathbb{1}_{a=x} - \pi_t(a))\)

Step 5: Plug In and Recover Eq. 2.12

Substituting the derivative into the expectation:

\(\dfrac{\partial \mathbb{E}[R_t]}{\partial H_t(a)} = \mathbb{E}\!\left[(R_t - \bar R_t)\,(\mathbb{1}_{a=A_t} - \pi_t(a))\right]\)

A single sample of this expectation is exactly the update from Slide 6:

\(H_{t+1}(a) = H_t(a) + \alpha\,(R_t - \bar R_t)\,(\mathbb{1}_{a=A_t} - \pi_t(a))\)

Conclusion: Eq. 2.12 is stochastic gradient ascent on \(\mathbb{E}[R_t]\).
Step 6: What the Derivation Bought Us
  • Expected update equals the true gradient \(\Rightarrow\) SGA convergence applies.
  • Baseline \(B_t\) is a free choice — affects variance, not expectation.
  • \(B_t = \bar R_t\) is simple and works well; it is not provably optimal.
  • Same recipe powers REINFORCE and policy gradient methods.
Why the Baseline Doesn't Bias the Update
Question: In the derivation we added \(-B_t\,\dfrac{\partial \pi_t(x)}{\partial H_t(a)}\) to each term in the gradient sum. Why is the expected update unchanged for any choice of \(B_t\) (constant, zero, or \(\bar R_t\))?
Because \(\sum_x \partial \pi_t(x)/\partial H_t(a) = 0\): multiplying by any \(B_t\) that doesn't depend on \(x\) and summing gives zero. Variance of the estimator changes; expectation does not.
The 10-Armed Testbed Shifted by +4
  • S&B re-run the testbed with all true means shifted up by +4: \(q_*(a) \sim \mathcal{N}(+4, 1)\).
  • Raw rewards are now uniformly positive — a stress test for the baseline.
  • Without a baseline, every \((R_t - 0)\) is positive — preferences will drift.

Choosing \(\alpha\): A Parameter Study
  • Like \(\epsilon\)-greedy and UCB, the gradient bandit has a sweet-spot \(\alpha\).
  • Too small \(\Rightarrow\) learns too slowly. Too large \(\Rightarrow\) noisy, unstable preferences.
  • S&B's parameter study shows the characteristic inverted-U curve.
Sutton & Barto, RL: An Introduction, Ch. 2, Fig. 2.6.
Question #4 — What If \(B_t = 1000\)?
Scenario: You replace the baseline \(\bar R_t\) with the constant \(B_t = 1000\) — much larger than any actual reward.

(a) Will the algorithm still converge to the optimal policy in expectation?
(b) What practical problem will you observe in your training curves?
(a) Yes — expected gradient is unchanged (Slide 17). (b) Variance explodes: every \((R_t - 1000)\) is large and negative, so each step makes a huge swing. Convergence will be slow and noisy.
From Nonassociative to Associative
  • All bandit settings so far: a single situation, find the best arm.
  • Real problems: the best action depends on context (time of day, user, sensor reading, screen colour).
  • We need a policy — a mapping from contexts to actions, not just one action.
The Coloured-Slot-Machine Story (S&B §2.9)
  • Several different \(k\)-armed bandit tasks; one shown each step at random.
  • The machine displays a colour indicating which task it is.
  • Goal: learn a policy — "if red, pull arm 1; if green, pull arm 2; …"
  • Reward depends on the (context, action) pair.
Sutton & Barto, RL: An Introduction, Ch. 2.9.
Contextual Bandit / Associative Search
Contextual bandit (associative search): at each step the agent observes a context \(C_t\), selects an action \(A_t\), and receives a reward \(R_t\) drawn from a distribution depending on \(C_t\) and \(A_t\). Goal: learn a policy \(\pi : \text{contexts} \to \text{actions}\) that maximises expected reward.
  • Like a bandit: action affects only the immediate reward.
  • Like full RL: requires learning a policy, not a single action.
  • Bridge problem between Lecture 1's bandits and the next lecture's MDPs.

Application Lens: Personalised Recommendations
  • Arms = candidate articles / ads / products.
  • Context = user features (location, time, history).
  • Reward = click / no-click, purchase / no-purchase.
  • Industry deployments (news feeds, ad targeting, A/B testing) are mostly contextual bandits — not full RL.
  • Why: feedback is immediate; no long-horizon planning needed (yet).
Question #5 — The Value of Context (Exercise 2.10)
A 2-armed bandit task whose true values switch between two cases:
  • Case A: \(q_*(1) = 10,\; q_*(2) = 20\) — probability 0.5
  • Case B: \(q_*(1) = 90,\; q_*(2) = 80\) — probability 0.5


(a) If you cannot tell which case you face, what is the best expected reward, and how should you act?

(b) If you are told which case (associative / contextual), what is the best expected reward, and how should you act?
(a) Marginal values: arm 1 → 50, arm 2 → 50. Indifferent; best expected reward = 50. (b) Pick arm 2 in case A (= 20) and arm 1 in case B (= 90) → 0.5·20 + 0.5·90 = 55. Context is worth +5 per step.

Bridge to Policy Gradient
  • Gradient-bandit derivation is a template for a much bigger family.
  • Replace softmax-over-preferences with softmax-over-neural-network-outputs \(\Rightarrow\) policy network.
  • Replace average-reward baseline with a learned value function \(\Rightarrow\) actor–critic.
  • Replace immediate reward with the return \(G_t\) \(\Rightarrow\) REINFORCE.
Exit Check & Next Lecture Bridge
  • Gradient bandit: preferences + softmax + stochastic gradient ascent.
  • Baseline reduces variance, never biases the expected update.
  • Contextual bandit: policy maps context → action; bridge to full RL.
  • All bandit settings share one feature: actions affect only the immediate reward.
  • Next: Markov Decision Processes — when actions also affect the next state.
Question #6 — Exit Check
Scenario: You are designing an algorithm for a news app that picks 1 of 5 headlines per visitor. You have rich user features (location, recent reads).

(a) Is this a bandit, contextual bandit, or full RL problem? Why?
(b) Would you reach for a gradient bandit here, or does the contextual nature change your choice? What would you change in the algorithm to make it context-aware?
(a) Contextual bandit — feedback is immediate (click / no-click) and there's no long horizon, but the best headline depends on user features. (b) Plain gradient bandit ignores context. Make preferences functions of context: \(H_t(a) = \theta_a^\top \phi(C_t)\). The same SGA derivation now gives a per-context policy update — the springboard to policy-gradient with function approximation.
