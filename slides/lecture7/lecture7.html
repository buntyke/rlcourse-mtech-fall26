Markov Processes

Markov Property

"The future is independent of the past given the present"

Definition
A state \(S_t\) is Markov if and only if \[ \mathbb{P}[S_{t+1} \mid S_t] = \mathbb{P}[S_{t+1} \mid S_1, \ldots, S_t] \]
Markov Property

"The future is independent of the past given the present"

Definition
A state \(S_t\) is Markov if and only if \[ \mathbb{P}[S_{t+1} \mid S_t] = \mathbb{P}[S_{t+1} \mid S_1, \ldots, S_t] \]
  • State captures all relevant information from the history
  • i.e., the state is a sufficient statistic of the future
State Transition Matrix

For a Markov state \(s\) and successor state \(s'\), the transition probability is:

\[ \mathcal{P}_{ss'} = \mathbb{P}[S_{t+1} = s' \mid S_t = s] \]

The state transition matrix \(\mathcal{P}\) collects all pairwise probabilities:

\[ \mathcal{P} = \begin{bmatrix} \mathcal{P}_{11} & \cdots & \mathcal{P}_{1n} \\ \vdots & \ddots & \vdots \\ \mathcal{P}_{n1} & \cdots & \mathcal{P}_{nn} \end{bmatrix} \]

Each row sums to 1.

Markov Process

A Markov process is a memoryless random process — a sequence of random states \(S_1, S_2, \ldots\) with the Markov property.

Definition
A Markov Process (or Markov Chain) is a tuple \(\langle \mathcal{S}, \mathcal{P} \rangle\)
  • \(\mathcal{S}\) is a (finite) set of states
  • \(\mathcal{P}\) is a state transition probability matrix, \(\mathcal{P}_{ss'} = \mathbb{P}[S_{t+1} = s' \mid S_t = s]\)

Markov Reward Processes

Markov Reward Process

A Markov reward process is a Markov chain with rewards.

Definition
A Markov Reward Process is a tuple \(\langle \mathcal{S}, \mathcal{P}, \mathcal{R}, \gamma \rangle\)
  • \(\mathcal{S}\) is a finite set of states
  • \(\mathcal{P}\) is a state transition probability matrix
  • \(\mathcal{R}\) is a reward function, \(\mathcal{R}_s = \mathbb{E}[R_{t+1} \mid S_t = s]\)
  • \(\gamma\) is a discount factor, \(\gamma \in [0, 1]\)

Concept of Returns

How do we measure the total reward accumulated from time step \(t\)?

Episodic (finite-horizon) return — when the task ends at time \(T\):

\[ G_t \doteq R_{t+1} + R_{t+2} + R_{t+3} + \cdots + R_T \]
Concept of Returns

How do we measure the total reward accumulated from time step \(t\)?

Episodic (finite-horizon) return — when the task ends at time \(T\):

\[ G_t \doteq R_{t+1} + R_{t+2} + R_{t+3} + \cdots + R_T \]

Discounted return — for infinite or continuing tasks:

\[ G_t \doteq R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \]

The discount \(\gamma \in [0,1]\) is the present value of future rewards.

  • Avoids infinite returns in cyclic Markov Reward Processes

Why Discount Rewards?
  • Avoids infinite returns in cyclic Markov Reward Processes
  • Uncertainty about the future may not be fully represented
  • If reward is financial, immediate rewards earn more interest than delayed rewards
  • Animal/human behaviour shows preference for immediate reward

\(\gamma \to 0\): myopic — only immediate reward matters
\(\gamma \to 1\): far-sighted — distant rewards matter equally

Recursive Return

Returns at successive time steps are related by a useful recursion:

\[ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots \] \[ = R_{t+1} + \gamma \bigl( R_{t+2} + \gamma R_{t+3} + \cdots \bigr) \]
\[ G_t = R_{t+1} + \gamma G_{t+1} \]

Holds for all \(t < T\); define \(G_T = 0\) at the terminal step.

Value Function
  • The value function \(v(s)\) gives the long-term value of state \(s\)
Definition
The state value function \(v(s)\) of an MRP is the expected return starting from state \(s\): \[ v(s) = \mathbb{E}[G_t \mid S_t = s] \]

Bellman Equation for MRPs

The value function decomposes into immediate reward + discounted future value:

\[ \begin{aligned} v(s) &= \mathbb{E}[G_t \mid S_t = s] \\ &= \mathbb{E}[R_{t+1} + \gamma R_{t+2} + \cdots \mid S_t = s] \\ &= \mathbb{E}[R_{t+1} + \gamma G_{t+1} \mid S_t = s] \end{aligned} \]
\[ v(s) = \mathbb{E}[R_{t+1} + \gamma v(S_{t+1}) \mid S_t = s] \]
Bellman Equation for MRPs

Expanding the expectation over successor states \(s'\):

\[ v(s) = \mathcal{R}_s + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}\, v(s') \]

Each successor \(s'\) is weighted by its transition probability \(\mathcal{P}_{ss'}\).

Student MRP: Bellman Equation Example
Bellman Equation in Matrix Form

The Bellman equation can be written concisely for all states simultaneously:

\[ \mathbf{v} = \mathcal{R} + \gamma \mathcal{P} \mathbf{v} \]

where \(\mathbf{v}\) is a column vector with one entry per state.

\[ \begin{bmatrix} v(1) \\ \vdots \\ v(n) \end{bmatrix} = \begin{bmatrix} \mathcal{R}_1 \\ \vdots \\ \mathcal{R}_n \end{bmatrix} + \gamma \begin{bmatrix} \mathcal{P}_{11} & \cdots & \mathcal{P}_{1n} \\ \vdots & \ddots & \vdots \\ \mathcal{P}_{n1} & \cdots & \mathcal{P}_{nn} \end{bmatrix} \begin{bmatrix} v(1) \\ \vdots \\ v(n) \end{bmatrix} \]
Solving the MRP Bellman Equation

The Bellman equation is a linear equation — it can be solved directly:

\[ \mathbf{v} = \mathcal{R} + \gamma \mathcal{P} \mathbf{v} \;\Rightarrow\; (\mathbf{I} - \gamma \mathcal{P})\,\mathbf{v} = \mathcal{R} \;\Rightarrow\; \mathbf{v} = (\mathbf{I} - \gamma \mathcal{P})^{-1} \mathcal{R} \]
  • Computational complexity is \(\mathcal{O}(n^3)\) for \(n\) states
  • Direct solution only practical for small MRPs
  • For large MRPs: iterative methods (e.g., dynamic programming, TD learning)

Markov Decision Processes

Markov Decision Process

A Markov decision process (MDP) is a Markov reward process with decisions.

Definition
A Markov Decision Process is a tuple \(\langle \mathcal{S}, \mathcal{A}, \mathcal{P}, \mathcal{R}, \gamma \rangle\)
  • \(\mathcal{S}\) is a finite set of states
  • \(\mathcal{A}\) is a finite set of actions
  • \(\mathcal{P}^a_{ss'} = \mathbb{P}[S_{t+1} = s' \mid S_t = s,\, A_t = a]\) — transition dynamics
  • \(\mathcal{R}^a_s = \mathbb{E}[R_{t+1} \mid S_t = s,\, A_t = a]\) — reward function
  • \(\gamma \in [0,1]\) — discount factor
Example: Student MDP
MDP Policy
Definition
A policy \(\pi\) is a distribution over actions given states: \[ \pi(a \mid s) = \mathbb{P}[A_t = a \mid S_t = s] \]
MDP Policy
Definition
A policy \(\pi\) is a distribution over actions given states: \[ \pi(a \mid s) = \mathbb{P}[A_t = a \mid S_t = s] \]
  • A policy fully defines the behaviour of an agent
  • MDP policies depend on the current state only — not the history
  • Policies are stationary (time-independent): \(A_t \sim \pi(\cdot \mid S_t),\; \forall t\)

Episodic vs Continuing Tasks

Episodic Tasks
  • Agent–environment interaction breaks naturally into episodes
    • Plays of a game, trips through a maze, repeated attempts
  • Each episode ends at a terminal state, then resets
  • Terminal time \(T\) is a random variable that varies per episode
  • Return is the finite sum: \[ G_t \doteq R_{t+1} + R_{t+2} + \cdots + R_T \]
  • State set extended: \(\mathcal{S}^+\) includes all non-terminal states plus the terminal state
Continuing Tasks
  • Interaction goes on continually without limit — no natural terminal state
    • On-going process control, long-running robot tasks
  • The finite-horizon return is problematic when \(T = \infty\): \[ G_t = R_{t+1} + R_{t+2} + \cdots \quad \to \quad \text{can diverge} \]
  • Solution: use the discounted return with \(\gamma < 1\): \[ G_t \doteq \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \]
  • If \(\gamma < 1\) and rewards are bounded, the sum is always finite
Why the Distinction Matters

Episodic

  • Finite \(T\), can use \(\gamma = 1\)
  • Return sums over one episode
  • Each episode is independent
  • Example: game of chess, maze run

Continuing

  • \(T = \infty\), requires \(\gamma < 1\)
  • Return must be discounted to converge
  • No episode resets
  • Example: robot life-span, process control

The return formula must match the task type — the choice of \(\gamma\) and \(T\) are linked.

Unified Notation

We can unify both cases with a single formula using an absorbing terminal state:

  • Terminal state transitions only to itself with zero reward
  • Episodic return extends to an infinite sum without changing the value
\[ G_t \doteq \sum_{k=t+1}^{T} \gamma^{k-t-1} R_k \] allowing \(T = \infty\) or \(\gamma = 1\) — but not both simultaneously

This single expression covers episodic and continuing tasks throughout the course.

Example: Pole-Balancing

Task: Keep a pole balanced on a moving cart. Failure = pole falls or cart runs off track.

Episodic framing:

  • Episode ends on failure; pole reset to vertical
  • Reward: \(+1\) every non-failure step
  • Return = steps until failure (can use \(\gamma = 1\))

Continuing framing:

  • No resets; task runs indefinitely
  • Reward: \(-1\) on failure, \(0\) otherwise
  • Requires \(\gamma < 1\) to ensure finite return

Both framings lead to the same optimal behaviour: balance the pole for as long as possible.

Summary
  • Markov Processes: memoryless random processes \(\langle \mathcal{S}, \mathcal{P} \rangle\) — the future depends only on the present state
  • Markov Reward Processes: Markov chains with rewards \(\langle \mathcal{S}, \mathcal{P}, \mathcal{R}, \gamma \rangle\) — defines returns and the value function \(v(s)\)
  • Markov Decision Processes: MRPs with actions \(\langle \mathcal{S}, \mathcal{A}, \mathcal{P}, \mathcal{R}, \gamma \rangle\) — the agent selects actions via a policy \(\pi\)
  • Episodic tasks have finite \(T\); continuing tasks require discounting (\(\gamma < 1\)) for convergence; unified via the absorbing-state convention
